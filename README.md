# Pop Salon

Pop Salon est un projet de logiciel de gestion pour **salons de coiffure indépendants**, pensé d’abord pour les **coiffeuses propriétaires qui travaillent seules**.

Le projet part d’un constat simple : les outils génériques de prise de rendez-vous représentent mal la réalité d’un salon de coiffure, notamment lorsque la durée dépend de la cliente, du service et des temps de pause liés à la coloration.

## Objectif du projet

Le POC vise à construire une première version centrée sur la **gestion des rendez-vous** :

- demandes de rendez-vous plus fluides ;
- durées par cliente et par service ;
- confirmations plus claires ;
- rappels automatiques ;
- meilleure prise en charge des cas de coloration et des plages parallèles.

Les modules d’inventaire, de tarification, de facturation et de comptabilité sont envisagés dans des phases ultérieures.

Projet en phase de cadrage / exploration produit.

## Contribuer

Le projet est encore au début, donc avant d’implémenter quoi que ce soit, l’idée est de garder une discipline simple :

1. lire `persona.md` et `prd.md` ;
2. discuter du besoin avant de coder une fonctionnalité importante ;
3. privilégier les contributions petites, ciblées et cohérentes avec le POC ;
4. éviter d’élargir le périmètre sans validation.

### Workflow suggéré

- créer une branche par sujet ;
- faire des commits petits et explicites ;
- ouvrir une PR avec le contexte, l’intention et les impacts ;
- documenter toute hypothèse produit importante.

## Principes du projet

- simplicité avant sophistication ;
- réalité métier avant agenda générique ;
- rapidité d’usage avant richesse fonctionnelle ;
- conformité avant bricolage comptable douteux.
